Secure your end to end communications using a Lets Encrypt SSL Certificate

Lets Encrypt SSL

A guide to deploying a free and trusted SSL certificate on your Node.JS app

Lets Encrypt is a free and trusted solution for your SSL needs. Throughout this guide, I will be covering an easy and quick method of setting up a SSL certificate for your Node.JS server. This guide has been tested to fully work on Ubuntu 16.0 and up servers, as well as on both Heroku and Digital Ocean droplets/dynos.

Open up the server's terminal and sign in with SSH keys or an username/password if needed.

First, make sure you either chmod your user account with sudo access, or run sudo before every command. Make sure you do this or else the certification client will not run.

Now we need to install some dependencies for LetsEncrypt

sudo apt-get install bc

Now we clone the Let's Encrypt repo to the server

sudo apt-get update

sudo git clone https://github.com/letsencrypt/letsencrypt /opt/letsencrypt

At this point, we will be connecting the SSL certificate to our domain name. Make sure the DNS settings of the domain are set up to point to your digital ocean droplet.

The domain should now be pointing to your digital ocean droplet, now we can generate a SSL certificate for use.

cd /opt/letsencrypt

./certbot-auto certonly --standalone

Follow the instructions presented on the CLI, you will have to provide an admin email address, specify domain names and agree to their Terms and Conditions.

Once you've finished that, you can now set up autorenewal for the Let's Encrypt SSL Certificate (every 90 days)

/opt/letsencrypt/certbot-auto renew

Now we get cron to handle automatic renewal. 

Run this command in the terminal to edit the server's cron jobs

sudo crontab -e

When the editor opens, add the following lines of code to the end of the file to tell cron to run the renewal command every Monday at 1 in the morning

00 1 * * 1 /opt/letsencrypt/certbot-auto renew >> /var/log/letsencrypt-renewal.log

30 1  * * 1 /bin/systemctl reload nginx

Congratulations! You've now added SSL to your Ubuntu Node.JS server, all end to end communication should be encrypted and authenticated.
